SNMP back list

Function

 

Check SNMP v1, the device can only process the information of v1. Check SNMP v2, the device can only process the information of v2. Check the SNMP v3, you can set user name,password and authentication type. The server needs to set the corresponding account when you want to access the device, it is for the security reasons. Right now the v1 and v2 are both null.

Community ：It is a string. It is a command between the manage process and the proxy process. It defined the authentication, access control and the management relationship between one proxy and one group of the managers. Please make sure the device and the proxy are the same.

Read Community ：The read community will read all the objects the SNMP supported in the specified name. The default setup is public.

Write Community ：The read community will read/write/access all the objects the SNMP supported in the specified name. The default setup is write.

Trap ：The SNMP trap is the information from the SNMP proxy to the administrator. The proxy can inform the administrator when an important event occurred or status has changed. SNMP v1 or SNMP v2 both supports.

Trap address : The destination address of the Trap information from the proxy program of the device.

Trap port : The destination port of the Trap information from the proxy program of the device. The default value is 162. It ranges from 1 to 65535.

SNMP port : It is the listening port of the proxy applications in the device. The default value is 161. The value ranges from 1 to 65535.
