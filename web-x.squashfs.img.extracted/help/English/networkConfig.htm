TCP/IP Introduce

TCP/IP mode is to set device login IP. It supports IPV4 and IPV6. IPV4 supports static IP and DHCP IP,IPV6 only supports static IP. After you set the IP address manually, Web interface can automatically go to the login interface of the new IP.

Parameter Function

Host Name It is to set current host device name. It max supports 15-digit character.
Ethernet Card Please note you can modify the default Ethernet port setup if there are several options.
Mode There are two modes: static mode and the DHCP mode.The IP/sub mask/gateway are null when you select the DHCP mode to auto search the IP. If you select the static mode, you need to set the IP/sub mask/gateway manually；If you select the DHCP mode, you can view the IP/sub mask/gateway from the DHCP. If you switch from the DHCP mode to the static mode, you need to reset the IP parameters.
MAC Address It is to display hose mac address (read-only).
IP Version It is to select IP version. IPV4 or IPV6. You can access the IP address of these two versions: http://[ fe80::5054:4cff:fefa:1682] 。
IPV6 newly added field includes：
MAC address： In the LAN, each host can get one unique MAC address. It is for you to access in the LAN. The address is read-only；
IP address: Just like the host of the IPv4. It is 128-digit and can not be null.
Subnet prefix： The number ranges from 0 to 128. The subnet prefix can mark a specified network MAC. It usually includes a structure of several layers.
Default gateway： Just like IPv4 gateway. It is 128-digit and can not be null.
Preferred DNS： Just lick the IPv4 DNS. It is 128-digit and can not be null.
Alternate DNS： Just lick the IPv4 DNS. It is 128-digit and can not be null.

Note：
1.It is for all the IPV6 address to check the validity. The IP address and the default gateway shall be in the same network section. That is to say, you need to set the subnet prefix field shall have the same length so that it can pass the verification.
2.When PPPOE is enabled, the IP address, subnet mask, default gateway of the IPV4 or IPv6 is read-only and can not restore the default setup.

Enable ARP/Ping set device IP address service

You can use ARP/Ping command to modify or set the device IP address if you know the device MAC address.

When it is enabled by default, during the reboot process, you can use the specified ping packet to set the device IP within 2 minutes. The service turns off after 2 minutes. The service also becomes off if you successfully set the IP address. The Ping function can not set the IP address if current function is disabled.
