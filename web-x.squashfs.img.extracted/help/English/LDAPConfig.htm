AD&LDAP Introduce

LDAP is so called Lightweight Directory Access Protocol. It is for the user rights verification when a device logs in. The users of different systems can share the account on the LDAP server to login the device. It is easy for the administrator to manage accounts and synchronize the accounts of different systems.

Server The server is to storage destination authorization user information and verification user information.
User Type Check the item and then select corresponding account on the login interface to login.
Port The default port is 389. The port value ranges from 1 to 65535. Usually we do not recommend value 1 to 1025 in case it conflicts with other port.
Base DN to begin searching users DN is user's only path on the LDAP domain. Set search user initial DN is to reduce user search time.
Anonymous Bind If the anonymous bind is checked, do not need to input binding DN and password.
Bind DN DN max has 64 characters. It can only contain number, letter, dot, equal sign, and underline.
Bind Password The password ranges from 0 to 32 characters. It can only contain number, letter.
LDAP Filter Specify DN conditions can filter redundancy information to quickly search user.
